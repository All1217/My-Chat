# [2026/4/6] 添加less相关依赖

根据你的 `package.json`，需要添加 Less 相关的依赖。Vite 原生支持 Less，只需要安装 Less 本身即可。

## 正确添加 Less 依赖

### 步骤1：安装 Less

在项目根目录执行：

```bash
npm install -D less
```

**注意**：
- 使用 `-D` (或 `--save-dev`) 因为 Less 是开发依赖
- **不需要**安装 `less-loader`（那是 Webpack 需要的）
- Vite 会自动处理 Less 文件的编译

### 步骤2：验证安装

安装成功后，`package.json` 的 `devDependencies` 中会新增：

```json
"devDependencies": {
  "@types/node": "^24.12.2",
  "@vitejs/plugin-vue": "^6.0.5",
  "@vue/tsconfig": "^0.9.1",
  "less": "^4.4.2",        // 新增
  "typescript": "~6.0.2",
  "vite": "^8.0.4",
  "vue-tsc": "^3.2.6"
}
```

### 步骤3：在 Vue 组件中使用 Less

**在 `<style>` 标签中指定 `lang="less"`：**

```vue
<template>
  <div class="container">
    <h1>Hello Less</h1>
    <button class="btn">按钮</button>
  </div>
</template>

<style scoped lang="less">
// 定义变量
@primary-color: #42b983;
@padding: 20px;

.container {
  padding: @padding;
  
  h1 {
    color: @primary-color;
  }
  
  // 嵌套语法
  .btn {
    background-color: @primary-color;
    color: white;
    padding: 10px 20px;
    
    &:hover {
      background-color: darken(@primary-color, 10%);
    }
  }
}
</style>
```

### 步骤4：也可以导入外部 Less 文件

**创建 `src/styles/variables.less`：**
```less
// 全局变量
@primary-color: #42b983;
@danger-color: #ff4444;
@font-size-base: 14px;
```

**在组件中导入：**
```vue
<style scoped lang="less">
@import '@/styles/variables.less';

.container {
  color: @primary-color;
  font-size: @font-size-base;
}
</style>
```

## 可选：配置全局 Less 变量（无需在每个文件导入）

如果需要全局使用变量，可以在 `vite.config.ts` 中配置：

```typescript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import path from 'path'

export default defineConfig({
  plugins: [vue()],
  css: {
    preprocessorOptions: {
      less: {
        additionalData: `@import "@/styles/variables.less";` // 全局注入
      }
    }
  },
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src')
    }
  }
})
```

配置后，组件中无需手动 `@import` 即可使用全局变量：

```vue
<style scoped lang="less">
.btn {
  color: @primary-color; // 直接使用，无需导入
}
</style>
```

## 常见问题

### 问题1：安装 Less 后报错
```bash
# 如果安装失败，尝试清除缓存重新安装
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
npm install -D less
```

### 问题2：TypeScript 类型提示
Less 不需要额外的类型声明，Vite 会处理。如果需要导入 `.less` 文件的类型，可以创建 `src/vite-env.d.ts`（如果不存在）：

```typescript
/// <reference types="vite/client" />

declare module '*.less' {
  const content: { [className: string]: string }
  export default content
}
```

### 问题3：想用 Less 的高级功能
Less 支持的所有功能都可以直接使用：

```less
// 混入（Mixins）
.border-radius(@radius) {
  border-radius: @radius;
}

// 函数
@base: #f04615;
@bg: @base + #333;

// 作用域
@var: red;
.page {
  @var: blue;
  color: @var; // blue
}
```

## 验证 Less 是否工作

1. 启动开发服务器：
```bash
npm run dev
```

2. 在任意 `.vue` 组件中添加带 Less 样式的代码
3. 浏览器中应该能看到样式生效，控制台无报错

**总结：只需执行 `npm install -D less` 一条命令即可！** Vite 的零配置特性让你不需要像 Webpack 那样配置 loader。

# [2026/4/6] 配置路由

根据你的 `package.json`，需要安装 Vue Router 并进行配置。以下是完整步骤：

## 步骤1：安装 Vue Router

在项目根目录执行：

```bash
npm install vue-router@4
```

**注意**：Vue 3 对应 Vue Router 4.x 版本

安装成功后，`package.json` 的 `dependencies` 会新增：
```json
"dependencies": {
  "vue": "^3.5.32",
  "vue-router": "^4.5.0"  // 新增
}
```

## 步骤2：创建路由配置文件

### 创建 `src/router/index.ts`

```typescript
import { createRouter, createWebHistory } from 'vue-router'
import type { RouteRecordRaw } from 'vue-router'

// 定义路由配置
const routes: RouteRecordRaw[] = [
  {
    path: '/',
    name: 'home',
    component: () => import('@/views/HomeView.vue')  // 懒加载
  },
  {
    path: '/about',
    name: 'about',
    component: () => import('@/views/AboutView.vue')
  },
  {
    path: '/chat',
    name: 'chat',
    component: () => import('@/views/ChatView.vue')
  }
]

// 创建路由实例
const router = createRouter({
  history: createWebHistory(import.meta.env.BASE_URL),  // HTML5 模式
  routes
})

// 可选：全局路由守卫
router.beforeEach((to, from, next) => {
  console.log('导航到:', to.path)
  next()
})

export default router
```

### 创建 `src/views/ChatView.vue`（聊天页面）

```vue
<template>
  <div class="chat">
    <h1>聊天室</h1>
    <div class="message-list">
      <div v-for="msg in messages" :key="msg.id" class="message">
        <strong>{{ msg.user }}:</strong> {{ msg.content }}
      </div>
    </div>
    <input 
      v-model="newMessage" 
      @keyup.enter="sendMessage"
      placeholder="输入消息..."
    />
    <button @click="sendMessage">发送</button>
  </div>
</template>

<script setup lang="ts">
import { ref } from 'vue'

interface Message {
  id: number
  user: string
  content: string
}

const messages = ref<Message[]>([
  { id: 1, user: '系统', content: '欢迎进入聊天室！' }
])
const newMessage = ref('')
let nextId = 2

const sendMessage = () => {
  if (newMessage.value.trim()) {
    messages.value.push({
      id: nextId++,
      user: '我',
      content: newMessage.value
    })
    newMessage.value = ''
  }
}
</script>

<style scoped lang="less">
.chat {
  padding: 20px;
  
  .message-list {
    border: 1px solid #ddd;
    padding: 10px;
    margin-bottom: 20px;
    height: 400px;
    overflow-y: auto;
    
    .message {
      margin-bottom: 10px;
    }
  }
  
  input {
    width: 70%;
    padding: 8px;
    margin-right: 10px;
  }
  
  button {
    padding: 8px 16px;
    background-color: #42b983;
    color: white;
    border: none;
    cursor: pointer;
    
    &:hover {
      background-color: darken(#42b983, 10%);
    }
  }
}
</style>
```

## 步骤4：在 main.ts 中注册路由

修改 `src/main.ts`：

```typescript
import { createApp } from 'vue'
import './style.css'
import App from './App.vue'
import router from './router'  // 导入路由

const app = createApp(App)

app.use(router)  // 注册路由插件
app.mount('#app')
```

```vue

```

## 步骤6：配置路径别名（可选但推荐）

修改 `vite.config.ts` 以支持 `@` 别名：

```typescript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import path from 'path'

export default defineConfig({
  plugins: [vue()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src')
    }
  }
})
```

同时修改 `tsconfig.json` 添加路径映射：

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"]
    }
  }
}
```

## 高级用法示例

### 动态路由（带参数）

```typescript
// router/index.ts 添加动态路由
{
  path: '/user/:id',
  name: 'user',
  component: () => import('@/views/UserView.vue'),
  props: true  // 将路由参数作为组件 props
}
```

```vue
<!-- UserView.vue -->
<template>
  <div>
    <h1>用户页面</h1>
    <p>用户ID: {{ id }}</p>
  </div>
</template>

<script setup lang="ts">
defineProps<{ id: string }>()
</script>
```

### 嵌套路由

```typescript
{
  path: '/chat',
  component: () => import('@/views/ChatLayout.vue'),
  children: [
    {
      path: 'room/:roomId',
      component: () => import('@/views/ChatRoom.vue')
    }
  ]
}
```

### 编程式导航

```vue
<script setup lang="ts">
import { useRouter, useRoute } from 'vue-router'

const router = useRouter()
const route = useRoute()

// 跳转到指定路径
const goToHome = () => {
  router.push('/')
}

// 带参数跳转
const goToUser = (id: number) => {
  router.push({ name: 'user', params: { id } })
}

// 获取当前路由参数
console.log(route.params.id)
</script>
```

## 常见问题

### 问题1：导入路径报错
确保已配置 `vite.config.ts` 中的 alias 和 `tsconfig.json` 中的 paths。

### 问题2：路由不显示内容
检查 `App.vue` 中是否有 `<router-view />` 组件。

### 问题3：404 问题
生产环境需要配置服务器支持 SPA，Vite preview 默认支持。

**总结**：Vue Router 4 为 Vue 3 提供了完整的路由解决方案，通过以上步骤即可为你的聊天应用添加多页面导航功能。
