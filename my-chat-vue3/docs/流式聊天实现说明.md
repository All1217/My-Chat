- 流式响应处理：逐块接收并显示

## 核心实现原理

### 流式请求处理
```typescript
// 使用fetch处理流式响应
const reader = response.body.getReader()
const decoder = new TextDecoder('utf-8')

while (true) {
    const { done, value } = await reader.read()
    if (done) break
    
    const chunk = decoder.decode(value, { stream: true })
    onMessage(chunk) // 实时回调更新UI
}
```

### 实时UI更新
```vue
<!-- 正在输入的AI消息 -->
<li class="message-ai" v-if="isStreaming">
    <div class="message-content">
        <MarkdownRenderer :content="streamingContent" />
    </div>
</li>

<!-- 实时更新内容 -->
streamingContent.value += chunk
scrollToBottom() // 每次更新都滚动
```

## 使用方式

### 启动项目
1. 确保后端服务运行在`localhost:8100`
2. 启动前端：`npm run dev`
3. 访问聊天页面：`/chat`

### 功能测试
1. 在输入框中输入问题
2. 按Enter或点击发送按钮
3. 观察AI回答的逐字显示效果
4. 可点击停止按钮中断生成

## 技术要点

### 1. 流式响应处理
- 后端返回`Flux<String>`流式数据
- 前端使用`TextDecoder`逐块解码
- 实时更新UI，类似DeepSeek官网效果

### 2. 自动滚动
- 使用`nextTick`确保DOM更新后滚动
- 监听消息列表容器的`scrollHeight`
- 每次新内容都触发滚动到底部

### 3. 状态管理
- `isStreaming`: 控制生成状态
- `streamingContent`: 累积流式内容
- `stopStreamingFn`: 取消请求函数

### 4. 错误处理
- 网络错误友好提示
- 请求取消不触发错误回调
- 生成中断时保存已有内容
